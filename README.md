# api-gateway

사내 **AI API Gateway** (`http://ai-api-gateway.tech.madup`) 의 **사용 튜토리얼과 사용 예시**를 모아두는 리포지토리입니다.

게이트웨이 기능마다 최상위 디렉토리가 하나씩 생기고, 그 안에는 실행 가능한 예제 스크립트와 친절한 설명 README가 함께 들어 있습니다. 처음 게이트웨이를 써보는 분이 1번 디렉토리부터 README를 읽으면서 예제를 돌려보면 어떤 식으로 API를 호출해야 하는지 익힐 수 있도록 구성했습니다.

---

## 📚 현재 포함된 튜토리얼

| 디렉토리 | 주제 | 대응 가이드 문서 | 시작점 |
|---|---|---|---|
| [`structured_output/`](./structured_output/) | Structured Output으로 LLM 응답을 JSON Schema에 맞춰 받기 | [`Structured_Output_Usage_Guide.md`](./Structured_Output_Usage_Guide.md) | [`structured_output/README.md`](./structured_output/README.md) |

앞으로 **텍스트 생성 외 기능**(이미지 생성, 임베딩, 툴 사용 등)의 튜토리얼이 추가되면 이 표에 한 줄씩 덧붙입니다. 새 디렉토리를 파고 README 를 쓰는 규칙은 아래 [기여 가이드](#기여-가이드)를 참고하세요.

---

## 공통 사전 준비

모든 튜토리얼이 공통으로 요구하는 것들입니다.

| 항목 | 기준 | 비고 |
|---|---|---|
| Python | 3.10 이상 | `match` 문과 `str \| None` 타입 힌트 사용 |
| 가상환경 | 권장 | `python -m venv .venv && source .venv/bin/activate` |
| API 키 | 필수 | 게이트웨이 담당자에게서 발급받아 각 튜토리얼의 `.env` 에 넣어주세요 |
| 네트워크 | 게이트웨이 접근 가능 | 사내망 / VPN / 방화벽 허용 |

세부 설치 과정(의존성, `.env` 설정)은 각 튜토리얼 디렉토리의 README에 단계별로 적혀 있습니다.

---

## 이 리포를 읽는 추천 순서

1. 이 README로 **어떤 튜토리얼이 있는지** 훑는다.
2. 관심 있는 주제의 디렉토리로 들어가 그 안의 README 를 읽는다.
3. README 상단의 "📍 README 맵" 으로 리포 전체 구조를 한눈에 보고, 궁금한 디렉토리는 링크로 바로 이동한다.

튜토리얼 디렉토리마다 내부 구조(챕터, 헬퍼, 템플릿 등)는 해당 디렉토리의 README 에서 트리로 보여주므로, 이 최상위 README에는 디렉토리 내부 트리를 싣지 않습니다.

## 기여 가이드

새로운 튜토리얼을 추가할 때 지켜주세요.

1. **최상위 디렉토리 이름**은 기능을 바로 알 수 있도록 명확히 (예: `structured_output`, `image_generation`).
2. **디렉토리 루트에 README.md 필수**. 대상 독자, 학습 로드맵, 실행 방법, 문제 해결 FAQ 를 포함합니다.
3. **예제 스크립트는 실제로 실행 가능**해야 합니다. 스크립트 상단 독스트링에 실행 명령을 꼭 적어 주세요.
4. **이 최상위 README의 "현재 포함된 튜토리얼" 표에 한 줄 추가**.
5. 가능하면 `common/` 패턴(설정 + 요청 헬퍼 분리)을 따라가면 다른 튜토리얼과 학습 곡선이 비슷해집니다.
